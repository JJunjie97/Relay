# USE 引擎内部通信协议 (ADR-001)

## 1. 概览与设计思想 (Overview)

**Universal Sequence Engine (USE)** 负责解析复杂的继电保护测试序列，并向下层 FPGA 驱动推流。
随着系统对时序极值（微秒级）和硬件资源（单片机与 FPGA 的 Ping-Pong 切换）压榨到了极限，本协议彻底抛弃了传统的 JSON 序列化/反序列化。

**核心设计准则**：
1. **零字典开销**：所有的通信指令、事件遥测均采用定长的“无键名”一维数组（Tuple）传输。
2. **纯平内存模型**：FSM 节点属性（Node Object）被完全打平（Flatten），没有任何嵌套层级，直接映射为底层的 C 语言 Struct 结构体。
3. **极致位运算（Bit-packing）**：所有高频度执行的控制字（DO、DDS波形、触发器判据）全部通过移位运算压缩至 `U32` 或 `U64` 类型内。
4. **前端算力转移**：解释层（Frontend Python）承担所有的波形预计算与控制字组装；底层引擎（Backend FSM）沦为一个**极速离散时间序列播放器**，只负责无脑的指针偏移和寄存器下发。

## 2. 内存映射空间与字典 (Macro Space & Dictionary)

为了在底层实现高效的 `Switch/Case` 路由分发，我们定义了极其工整的内存映射空间。

### 2.1 指令与属性键映射 (Hex IDs)

| 分区 | 宏名称 | ID | 负载数据类型 / 描述 |
|---|---|---|---|
| **0x00 区间**<br>*(引擎控制指令)* | `CmdStart` | 0x01 | **负载:** `[U16, List[Dict]]` 硬件防抖时间(ms)与节点数组。空转引擎并设置防抖。 |
| | `CmdStop` | 0x02 | **负载:** 无（仅命令字）。紧急停止、切断输出、FSM 复位。 |
| | `CmdManualTrig`| 0x03 | **负载:** `[I16]` 目标节点ID。强制跨节点跳转。 |
| | `CmdUpsertNodes`| 0x04 | **负载:** `[List[Dict]]` 节点对象数组。在线热更新/插拔节点。 |
| **0x10 区间**<br>*(节点属性结构)* | `Id` | 0x10 | `[I16]` 当前节点的唯一标识。 |
| | `DoActions` | 0x11 | `[List[U32]]` 动作序列。格式见 *[第五章 5.1]*。 |
| | `Static` | 0x12 | `[List[U64]]` 稳态波形 DDS 控制字。格式见 *[第五章 5.3]*。 |
| | `Interval` | 0x13 | `[U32]` 暂态步进/更新周期时间 (ms)。 |
| | `Steps` | 0x14 | `[List[List[U64]]]` 非均匀暂态波形步进矩阵，按 Tick 偏移。 |
| | `PhaseGate` | 0x15 | `[U64]` 相控切换配置。`(Channel << 32) | Phase`。 |
| | `ResetTime` | 0x16 | `[U32]` 节点跳转间的复归态维持时间 (ms)。 |
| | `ResetStatic` | 0x17 | `[List[U64]]` 复归态期间的 DDS 控制字。 |
| | `ResetDo` | 0x18 | `[U32]` 复归态边界 DO 掩码组合。格式见 *[第五章 5.2]*。 |
| | `Triggers` | 0x19 | `[List[U64]]` 条件反射触发器阵列。格式见 *[第五章 5.4]*。 |
| **0x20 区间**<br>*(引擎遥测事件)* | `EvtValueUpdate`| 0x21 | 引擎主动控制的状态更新 (id/tick/DO)。 |
| | `EvtTrigger` | 0x22 | FSM 条件判据被满足时的触发瞬态快照事件。 |
| | `EvtDiChange` | 0x23 | 硬件 DI 输入变更事件（独立于引擎生命周期）。 |

### 2.2 触发条件枚举值 (Values)
*注：以下枚举不是用于路由的 Key，而是打包在 `Triggers` (0x19) U64 内部的纯粹 `Condition` 数值。*

| 宏名称 | 值 | 描述 |
|---|---|---|
| `ConDiMatch` | 0 | DI 输入硬件匹配（带硬件层滤波掩码） |
| `ConTimeout` | 1 | 驻留时间超时自动跳转 |
| `ConCountOver` | 2 | 限定循环次数/Tick 耗尽自动跳转 |

## 3. 通信操作帧与遥测帧 (Communication Frames)

通信信封彻底遵循了定长数组（Tuple）交互，拒绝 JSON 字典格式。

### 3.1 下发引擎操作指令 (Interpreter -> Engine)
所有的下发指令必须以 `0x00 区间` 的 `Cmd` 宏作为 Tuple 的首个元素。

```python
# 1. 硬件外设初始化并启动空转
# (命令ID, 去抖动目标值)
(CmdStart, 61)

# 2. 写入波形或运行中热更新 (Hot Reloading)
# (命令ID, 节点对象数组)
(CmdUpsertNodes, [Node1, Node2, ...])

# 3. 手动干预流转 / 跳转首节点启动
# (命令ID, 目标节点ID)
(CmdManualTrig, 0)

# 4. 灾难停止/复位
# (命令ID,)
(CmdStop,)
```

### 3.2 引擎状态高频遥测 (Engine -> Interpreter)
底层向前端的高频推送必须使用无字典的一维数组。首个元素必为 `0x20 区间` 的事件类型。

```python
# 1. 值更新事件 (EvtValueUpdate) — 每次 Up/PhaseGate 生效后发送
# [事件ID, 当前节点ID, 逻辑步数Tick, 硬件时间戳]
# *注*: Tick = -1 代表复归态(Reset)。nodeId 变化即表示触发跳转，无需单独触发事件。
[EvtValueUpdate, 1, 2, 123456]

# 2. DO 变更事件 (EvtDoChange) — SpawnDo 定时器循环内发送
# [事件ID, DO掩码, 硬件时间戳]
[EvtDoChange, 0x03, 123456]

# 3. DI 变更事件 (EvtDiChange) — 硬件外部事件，独立于引擎生命周期
# [事件ID, DI状态, 硬件时间戳]
# *注*: 始终转发到前端显示 + handler 逻辑，无论引擎是否运行。
[EvtDiChange, 1, 123456]
```

## 4. 核心数据模型 (Core Data Model - The Node)

以下定义了底层真正缓存和解析的内存数据结构（即 `Node Object`）。经过展平优化，它是没有任何嵌套的纯 1 维 `Flat Struct`。
*注意：所有的延时/时间单位均为毫秒 (ms)。*

```python
{
  # 基础身份与稳态维持
  "Id": 1,                          # [I16] 节点句柄
  "Static": [
    0x0000006400000000              # [U64] 必须包含基波，谐波可叠加。见 [5.3]
  ],
  "DoActions": [
    327681                          # [U32] 附带延时的执行动作。见 [5.1]
  ],
  
  # 暂态步进控制 (可选)
  "Interval": 100,                  # [U32] 步进滴答(Tick)之间的时间间隔
  "Steps": [
    [ 0x0000006400000000 ],         # [List[U64]] 暂态波形步进矩阵，二维数组设计，
    [ 0x003FFFFB00000000 ]          # 引擎按 Tick 索引读取一维列进行推流
  ],
  
  # 硬件级门控 (可选)
  "PhaseGate": 0x0000000040000000,  # [U64] (Channel << 32) | Phase
  
  # 节点跳转间的复归态/复归态 (可选)
  "ResetTime": 100,                 # [U32] 若存在且 > 0，则该节点前后存在空闲复归态
  "ResetStatic": [
     0x0000000000000000             # [U64] 复归态期的默认挂起波形。见 [5.3]
  ],
  "ResetDo": 327683,                # [U32] 复归态出/入边界的 DO 控制。见 [5.2]
  
  # 状态机跳转触发网络
  "Triggers": [
    0x0000FFFF00000000              # [U64] 极致压缩的条件反射弧。见 [5.4]
  ]
}
```

## 5. 位运算打包规范 (Bitwise Packing Protocols)

这部分详细定义了上述模型中 32 位和 64 位整数究竟是如何通过移位运算（Bit Shift）组合而成的物理意义。

### 5.1 DO 动作打包 (`DoActions`)
类型：`[U32]`
格式：`(Delay_ms << 16) | (Mask & 0xFFFF)`
* **`Delay_ms` (高 16 位)**：表示自**进入该节点起**，延迟多少毫秒后执行此 DO 动作。0 表示进入节点瞬间立即执行。
* **`Mask` (低 16 位)**：覆盖到物理 DO 硬件开关量寄存器上的掩码值。

### 5.2 复归态 DO 掩码打包 (`ResetDo`)
类型：`[U32]`
格式：`(ExitMask << 16) | (EnterMask & 0xFFFF)`
* **`EnterMask` (低 16 位)**：在刚刚**进入复归态时**瞬间，立刻输出至硬件的 DO 掩码。
* **`ExitMask` (高 16 位)**：在复归态耗尽，即将**退出复归态（并跳入下一个业务节点）前**瞬间，输出至硬件的 DO 掩码。

### 5.3 DDS 载波控制字打包 (`Static`, `Steps`, `ResetStatic`)
类型：`[U64]`
此格式将通道、频层、幅度和相位无缝封装送入 FPGA：
* **Bits [63:60]** (4 位)：`Channel ID` 通道号 (0~15)
* **Bits [59:54]** (6 位)：`Layer ID` 谐波层级 (0=DC直流偏置, 1=基波, 2~63=各次谐波)
* **Bits [53:32]** (22 位)：`Amp` 幅度/递增步长。带符号 22 位补码。底层引擎在累加时，高 16 位给 FPGA 作为精准 I16 物理幅度，低 6 位留作底层内部高精度小数步进累加。
* **Bits [31:0]**  (32 位)：`Freq / Phase` 频率或相位原生控制字。

> 组装公式: `U64 = (((ch & 0xF) << 6) | (layer & 0x3F)) << 54 | ((Amp & 0x3FFFFF) << 32) | (Freq & 0xFFFFFFFF)`

### 5.4 FSM 条件触发器打包 (`Triggers`)
类型：`[U64]`
所有的触发判据和掩码判定被折叠为单条 64 位指令：
* **Bits [63:48]** (16 位)：`Condition` 触发条件类型数值 (0=`ConDiMatch`, 1=`ConTimeout`, 2=`ConCountOver`)。
* **Bits [47:32]** (16 位)：`NextId` 条件满足后即将跳转的下一节点 ID (0~0xFFFE)。0xFFFF 表示终止 FSM。
* **Bits [31:0]**  (32 位)：`UnionData` 联合判定参数。
  * 若类型为**超时(1)**: 填入超时的毫秒数 (ms)。
  * 若类型为**DI匹配(0)**: 填入极其复杂的硬件滤波掩码配置：
    * `Bits [0:7]`: 需要监听的物理 DI 通道匹配掩码。
    * `Bit 8` (0x100): 逻辑模式 (0=OR, 任意通道符合即触发; 1=AND, 全部通道符合才触发)。
    * `Bit 9` (0x200): 参考系 (0=FPGA 收到 Start 后返回的初始 DI，可作为首个状态前的基准; 1=相对于上一个状态的 DI 快照)。
    * `Bit 10` (0x400): 极性检测 (0=翻转/变位即触发; 1=恢复参考系基准原始态即触发)。

> 组装公式: `U64 = (Condition << 48) | ((NextId & 0xFFFF) << 32) | (UnionData & 0xFFFFFFFF)`

### 5.5 相控门控配置打包 (`PhaseGate`)
类型：`[U64]`
用于精准的过零点或特定相位切换控制：
* **Bits [63:32]** (32 位)：`Channel` 门控参考通道号（U16）。
* **Bits [31:0]**  (32 位)：`Phase` 相控切换的目标相位控制字（U32）。

> 组装公式: `U64 = (Channel << 32) | Phase`
