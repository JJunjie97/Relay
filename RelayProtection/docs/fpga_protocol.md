## 1. 参数解析规则

> [!CAUTION]
> **不同层级 (Index) 的参数物理意义完全不同。**

向 FPGA 写入波形参数时，帧负载包含两个 32 位数据 `p1` 和 `p2`，其含义取决于目标层级 (Index)：

| 层级 | p1 | p2 |
|------|----|----|
| 交流层 (Index `0x00` ~ `0x3F`) | 幅度 (Amplitude) | 相位 (Phase) |
| 直流层 (Index `0xFF`) | 幅度 (Amplitude) | 频率 (Frequency) |

### 数据截断规则

- **p1（幅度）**：上位机传入 `U32`，协议层仅取**高 16 位**发给 FPGA。FPGA 将其作为 `I16`（有符号 16 位整数）处理。因此幅度寄存器值不能超过 32767，否则会被解读为负数。
- **p2（频率/相位）**：
  - 直流层（频率）：FPGA 取完整 32 位 `U32`。
  - 交流层（相位）：FPGA 仅截取**高 16 位**作为 `U16` 进行环形累加（满 `0xFFFF` 绕回 `0x0000`）。

## 2. 三级缓冲区架构

FPGA 内部采用 Base → Shadow → Active 三层流水线缓冲，保证波形切换平滑、多通道更新无毛刺。

```
┌─────────────────────────────────────────────────┐
│              FPGA 寄存器三级缓冲区                │
├─────────────────────────────────────────────────┤
│ Base   (基准缓冲)  ─ STEP 运算的累加锚点         │
│ Shadow (影子缓冲)  ─ 预加载区，等待翻转生效       │
│ Active (物理输出)  ─ 直接驱动 DAC 实时输出        │
├─────────────────────────────────────────────────┤
│ SYS_UPDATE 触发时 Active ↔ Shadow 瞬间互换指针   │
│（乒乓翻转 / Ping-Pong Buffer Flip）              │
└─────────────────────────────────────────────────┘
```

- **Base**：所有写入操作首先覆盖 Base。STEP 运算读取 Base 作为被加数，结果写回 Base 并同步到 Shadow。
- **Shadow**：预加载区。`WR_STAGE` 和 `STEP_STAGE` 仅写 Shadow 而不动 Base，用于隔离场景（如 Mode 3 的复归态）。
- **Active**：物理输出寄存器，只能通过指令驱动的翻转（Update）来更新。
- **SYS_SYNC**：将当前 Active 层数据反向覆写 Base 和 Shadow，实现三区完全对齐。用于消除乒乓翻转后 Shadow 中残留的脏数据。

## 3. 指令集

硬件支持两种帧格式：**SYS 帧**（`0x5A` 头，4 字节）和 **PARAM 帧**（`0xA5` 头，12 字节）。

### 3.1 ACK 机制

> [!IMPORTANT]
> FPGA 对每一条物理帧（SYS 或 PARAM）在执行完毕后回复一个 ACK 帧（含硬件时间戳）。`USEEngine` 依靠 `sentCount` 与 `ackCounter` 的绝对一致性来确认命令完成进度。

### 3.2 系统指令 (SYS Commands)

| 代码 | 常量名 | 说明 |
|:-----|:-------|:-----|
| `0x00` | `SYS_START` | 启动波形发生器输出 |
| `0x01` | `SYS_STOP` | 停止发生器输出，输出置零 |
| `0x04` | `SYS_RESET` | 全局硬件硬复位，清空所有缓冲区 |
| `0x05` | `SYS_UPDATE` | 触发乒乓翻转：Active ↔ Shadow 指针互换 |
| `0x06` | `SYS_SYNC` | 三区对齐：将 Active 反向写入 Base 和 Shadow |
| `0x20` | `SYS_SET_DBNC` | 设置 DI 防抖滤波时间（`p_U8`，每单位 ≈ 0.328ms @ 50MHz） |
| `0x21` | `SYS_SET_DO` | 写入数字输出 (DO) 状态掩码 |

### 3.3 DDS 写入指令 (Write Commands)

当前代码实现的指令：

| 代码 | 常量名 | 写入目标 | 自动翻转 |
|:-----|:-------|:---------|:---------|
| `0x10` | `DDS_WR_SHADOW` | Base + Shadow | 否 |
| `0x11` | `DDS_WR_STAGE` | 仅 Shadow | 否 |

协议保留指令（代码中未使用）：

| 代码 | 说明 |
|:-----|:-----|
| `0x12` | 写 Base + Shadow 后自动触发翻转 |
| `0x13` | 写 Shadow 后自动触发翻转 |

### 3.4 DDS 步进指令 (Step Commands)

当前代码实现的指令：

| 代码 | 常量名 | 运算规则 | 自动翻转 |
|:-----|:-------|:---------|:---------|
| `0x14` | `DDS_STEP_SHADOW` | Base += Step → 结果写入 Shadow | 否 |
| `0x15` | `DDS_STEP_STAGE` | Shadow += Step（Base 不变） | 否 |

协议保留指令（代码中未使用）：

| 代码 | 说明 |
|:-----|:-----|
| `0x16` | Base += Step → 结果写入 Shadow 后自动触发翻转 |
| `0x17` | Shadow += Step 后自动触发翻转 |

### 3.5 相位门控指令

| 代码 | 常量名 | 说明 |
|:-----|:-------|:-----|
| `0x1F` | `DDS_PHASE_GATE` | 延迟翻转至指定通道达到目标相位角时自动执行 |

> [!CAUTION]
> **`DDS_PHASE_GATE` 的参数结构与其他 PARAM 指令完全不同：**
>
> - **`chMask` 字段**：不是通道掩码，而是 FPGA 内部计算循环的匹配索引。FPGA 运行 1024 步计算循环（16 通道 × 64 次谐波），传入值必须是 `物理通道号 × 64`（左移 6 位）。代码中由 `BuildPhaseGateFrame(physical_ch, phase_u32)` 自动处理。
> - **`p2` 字段**：取高 16 位作为目标相位比较值。
>
> **FIFO 阻塞效应**：如果目标相位尚未到达，该指令会阻塞在 FPGA 的 FIFO 头部不返回 ACK。上位机可以将后续指令（如 `SYS_SYNC`、下一步步进帧）紧跟在门控帧后一次性发出，利用 FIFO 阻塞实现硬件级自动排队。

## 4. 步进模式的溢出行为

STEP 指令的加法在 FPGA 内部因数据类型不同而有不同的越界处理：

| 参数 | 数据类型 | 溢出策略 |
|:-----|:---------|:---------|
| 频率 (p2, 直流层) | U32 + I32 | 饱和（到达上限后维持最大值） |
| 相位 (p2, 交流层) | U16 + I16 | 环形绕回（`0xFFFF` + 1 = `0x0000`） |
| DC 幅度 (p1, 直流层) | I16 + I16 | 饱和 |
| AC 幅度 (p1, 交流层) | U16 + I16 | 环形绕回 |

> [!WARNING]
> **AC 层幅度溢出是最危险的场景**：Base 中幅度以 `U16` 存放和相加，但 DAC 运算时以 `I16` 读取。当累加结果从 32767 跨到 32768 时，`I16` 解读为 -32768，导致波形幅值瞬间反转。上位机在发送步进指令前必须对累加结果做边界检查和裁剪。
